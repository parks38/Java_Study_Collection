* 비식별자 / 식별자 
* ## ❤️ 식별자/ 비식별자

- 부모의 PK 속성이 자식의 PK 속성 or 일반 속성으로

**식별자**

- PK 속성의 숫자가 증가할 수록 SQL 구문이 복잡해져 `복잡성`으로 개발 오류 유발

**비식별자** 

- 테이블 간의 과다한 조인 유발으로 `조인에 의한 성능 저하`

⭐  업무적 특징, 조인 관계 , PK 구성을 고려하여 비식별자/ 식별자 결정 

 **[PK 속성이 너무 많은 경우]**

 : 비슷한 유형의 테이블을 조인하면 너무 많은 조인 항목 생성 

  - 식별자와 비식별자를 조정하지 않고 식별자 관계만 연결하면 다수의 PK 증가

**[식별자 관계로만 설정]**

 : 부모 테이블에는 PK 속성 수가 1개이고 자식 테이블의 PK 속성 수는 부모 테이블 PK 속성 수 + 1

1개이상의 속성이 추가되어야 1:M 관계를 만족할 수 있습니다.

원 부모테이블 : 1개

2대 부모테이블 : 2개 이상 = 원부모 1개 + 추가 1개 이상

3대 부모테이블 : 3개 이상 = 원부모 1개 + 2대 1개 + 추가 1개 이상

4대 부모테이블 : 4개 이상 = 원부모 1개 + 2대 1개 + 3대 1개 + 추가 1개 이상

5대 부모테이블 : 5개 이상 = 원부모 1개 + 2대 1개 + 3대 1개 + 4대 1개 + 추가 1개 이상

⭐  PK 속성이 지속적으로 증가하여 개발 복잡성과 오류 가능성 유발

- 유일성 : 모든 인스턴스 유일히 구분
- 최소성 : 유일성을 만족하는 최소의 수
- 불변성 : 식별자의 값은 변하지 않아야 함.
- 존재성 : 데이터 값이 존재해야 함 (null 값 안됨)

**[비식별자 관계로만 설정]**

 : 부모 엔티티타입에 있는 PK 속성으로부터 상속되어 자식 엔티티타입에 존재하는 경우 많음. 

  - entity type 을 비식별자 관계 : 속성 조회 시 자식 테이블로 상속되지않아 쓸데없이 부모 테이블 까지 찾아가야 하는 경우
